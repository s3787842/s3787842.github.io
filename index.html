Project Idea

My project idea is an application or program that allows users to calculate how many blocks or bricks would be required to create a structure, be it a larger building or just a house. The user would enter the necessary details such as the dimensions of the building, whether bricks or blocks are being used and the size of the materials. Things such as doors and windows would have to be accounted for by the user and in order to use the application and the entered details would have to be very accurate and specific. Once this is finished, the application will calculate and give an estimate on how many resources would be required.

The reasoning behind choosing this project is because my father is a bricklayer and he is the one that suggested it to me. He is also the foreman of his worksites and is the one who must make the orders for materials for the current job. While there are programs that would allow for the calculations to be done, none of them are easy to use and require a lot of effort to use them for this purpose. Therefore, I believe this project would be useful and has a market that would use it.

The main feature of this product would be its ability to give an accurate estimate on the number of blocks or bricks required to design a specific structure. The application would request user input and then it would make these calculations based on the user's responses. These details would include whether blocks or bricks are being used, all the required dimensions for the structure including door frames, windows and all other relevant areas where the blocks would not be going. After recieving all these details, the application would then make an accurate estimate as to the required resources. It would be critical to alert the user that this is just an estimate as there are many different scenarios that could affect the outcome and cause inaccuracies that are unavoidable. Another feature of the program could be a visual showing off what the machine believes the building will look like. It could create a model based off the previously entered dimensions and show the users what the final outcome may resemble. This may also help user's identify mistakes in their plans or when details have been incorrectly entered. In order to do this, the hardware running the application would have to be powerful which may make this part of the application not entirely plausible as it may be running on less powerful hardware, such as phones, and will not be able to produce a good result, causing more issues then good for the user. The application could also potentially be able to not only account for blocks and bricks but also have options to give an estimate on other required materials. This would include glass for windows, the material for door and window frames and even possibly floorboards or benches. As this application is designed specifically for bricklayers, implementing these features may not provide useful to them but will open up the possibility for other professions to use the program such as carpenters or even just the general public who are attempting to build their own property. With this feature, there could also be an option for users to have a selection of different materials that the user desires and it would be able to list prices and compare between different stores locally and online. In order to execute this, agreements with local hardware and other stores would be required. As they may not agree until the application is larger and they can see a demand for this, this feature could be left and implemented into the design later if the need or want arises from users. These are my current ideas for the application that I believe would be highly beneficial but many more may arise as development progresses and the need for certain features is discovered.

To develop the software for this application, a modelling program would be a prerequisite. AutoCAD is what I believe would be the best solution as I have previous experience using it and have a good understanding of its capabilities and am certain it is an appropriate tool. The different sizes of blocks and bricks would already be implemented into the program and AutoCAD could model the structure using the dimensions and the dimensions of the blocks already coded into the program. This would give an accurate estimate of the amount required for development of a project and also benefits the idea of showing a model to the user of what the finished structure may resemble.

In order to begin creating this project, I would need to have an understanding of all the modelling programs available to me and also even obtain the skills required to write the software myself if existing programs such as AutoCAD do not have the capabilities required. If the application is put onto phones, a background in phone app development will also be necessary to create the project. Knowledge on databases would also be of great benefit as implementing options such as different materials and pricing would need to be stored on a database that can be fetched whenever necessary.

If this project is to succeed, I believe it could have a noticeable impact on the way worksites are ran and make developing structures much easier and efficient. The amount of wasted resources would also be reduced as excess materials will no longer be ordered, which in turn will also save money. Cases where not enough resources were ordered will also no longer exist, saving large amounts of time and speeding up developments. These are the reasons why I believe that my project idea is a worthwhile investment and would have a beneficial impact on the way not only companies run their worksites, but also on the way smaller businesses and everyday people design their own structures.
